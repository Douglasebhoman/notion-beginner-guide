## Writing Approach

Each document in this repository is written with a specific reader in mind — not a general "beginner," but a person with a concrete goal they are trying to accomplish. Structure follows user intent:

1. What does this person need to *do*?
2. What do they need to *know* before they can do it?
3. What might go wrong, and how do they recover?

This approach reflects the documentation philosophy I bring to all technical writing work.
